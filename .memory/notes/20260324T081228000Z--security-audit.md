Summary:

- Used the `find-skills` workflow to identify `trailofbits/skills@semgrep` as the strongest general-purpose security audit skill candidate for this repo, based on install count and source reputation.
- Performed a manual security review of the Hawk-Eye client/runtime and Vite plugin write path, then supplemented it with `pnpm audit --json` and the existing Vitest suite.

Primary findings:

1. Same-page JavaScript can escalate into local source-file writes when `enableSave` is enabled.
   - The inspector mounts into an open shadow root, registers global document click handlers, and does not distinguish trusted user gestures from synthetic events.
   - A hostile script running in the inspected app can drive Hawk-Eye's own UI flow to select signed elements, mutate draft controls, and trigger `Update Design`, which turns an in-page XSS into source mutation on the developer machine.
   - Relevant files: `packages/client/src/DesignTool.tsx`.

2. Save capabilities are reusable bearer tokens that are not bound to a specific websocket connection.
   - The server reuses the same capability for a `clientId`, validates only the `(clientId, capability)` pair, and explicitly accepts reuse after websocket identity changes.
   - The browser client stores the `clientId` on `globalThis`, which makes replay easier for any same-page script that can observe or replace it.
   - Relevant files: `packages/vite-plugin/src/plugin-state.ts`, `packages/vite-plugin/src/save-handler.ts`, `packages/client/src/ws-client.ts`, `tests/save-handler.test.ts`.

Secondary findings:

- `pnpm audit --json` reported three tooling advisories on 2026-03-24:

  - `eslint > file-entry-cache > flat-cache > flatted 3.3.4`: CVE-2026-32141, high, patched in `flatted >= 3.4.0`
  - `eslint > file-entry-cache > flat-cache > flatted 3.3.4`: CVE-2026-33228, high, patched in `flatted >= 3.4.2`

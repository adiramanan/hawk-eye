## Context

- Investigated a UI bug where switching from each-side editing to grouped all-sides editing only changed the visible field state and did not actually propagate the displayed value until the user typed again.

## Actions

- Updated the per-side toggle handler in `packages/client/src/controls/PerSideControl.tsx` so switching into grouped mode immediately commits the currently displayed grouped value to the affected sides.
- Added a regression test covering the all-sides toggle propagation behavior.
- Re-ran the targeted new regression and the focused design-tool suite.

## Decisions

- Toggling into grouped mode should be a semantic commit, not a display-only mode change.
- The grouped field should use the visible top/vertical/horizontal display value as the source of truth when propagating to the other sides.

## Open Threads

- The same immediate-commit behavior is now implemented for both `all-each` and `opposite-each` groupings, but only the all-sides path is currently covered by an explicit regression test.

## Summary

Linking per-side controls now applies the currently shown grouped value across the linked sides immediately, without requiring a follow-up manual edit.

## Context

- After the initial draft rebase fix, the panel could still lose class-target initialization during reselection because `ensureDraftForMeasurement(...)` rebased against placeholder pre-analysis details with empty `classTargets`.
- That meant the selected class-target dropdown and class-backed baselines could disappear or reset before the next style-analysis payload arrived.

## Actions

- Updated `packages/client/src/drafts.ts` so `rebaseSelectionDraft(...)` preserves the last resolved class context when new selection details are only placeholder pre-analysis values.
- Kept the draft flagged as `styleAnalysisResolved: false` for pending-analysis state while still using the preserved class context to initialize baselines.
- Added a unit regression in `tests/drafts.test.ts` covering the preserved class-context behavior during a pending analysis refresh.
- Added a `DesignTool` regression in `tests/design-tool.test.ts` covering reselection of the same class-target element before fresh analysis returns.
- Re-ran `pnpm vitest --run tests/drafts.test.ts tests/design-tool.test.ts` and confirmed all focused tests passed.

## Decisions

- Placeholder selection details (`styleAnalysisResolved: false`, empty `classTargets`) should not overwrite an already resolved class-target context during rebase.
- Pending-analysis UI state is preserved separately from the property snapshots, so the panel can stay correctly initialized while the footer still knows fresh analysis is outstanding.

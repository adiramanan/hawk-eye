## Context

- The inspector currently renders properties from a global V1 allowlist plus element-context heuristics.
- Class-target selection chooses which selector preview/save applies to, but the panel does not yet know which properties a selected authored class target actually declares.

## Actions

- Extended `AuthoredClassTarget` with both `declaredPropertyIds` and raw `declaredCssValues`, and taught the stylesheet index to derive both from each authored rule body, including practical shorthand expansion for `padding`, `margin`, `background`, `border`, `border-radius`, and `font`.
- Kept the inspector groups visible instead of hiding them; moved the class-target behavior into draft initialization so the selected class target preloads only its own authored values while non-declared properties remain available but uninitialized.
- Updated the style-analysis merge path to rebuild a clean draft from the selected class target as soon as authored class-target metadata arrives, so the first selected class initializes correctly without requiring a manual dropdown switch.
- Preserved detached mode as the full element-level view; detaching still drops back to the existing combined element snapshots instead of class-authored initialization.
- Added payload and UI regressions for declared-property metadata, class-specific initialization with all groups still visible, detached restore behavior, and the previously fixed class-target switch safety.
- Verified the work with `pnpm vitest --run tests/ws-server.test.ts tests/design-tool.test.ts tests/smoke.test.ts tests/save-handler.test.ts` and checked the touched files with `ReadLints`.

## Decisions

- Use inspector property ids plus raw CSS declaration values as the class-target contract: ids are used to understand which authored properties exist, while raw CSS values let the client initialize the selected class target from authored declarations instead of merged computed styles.
- Keep all existing property groups visible in class-target mode and only change initialization semantics, so the user can still add new declarations to the selected class without switching to detached mode first.

## Open Threads

- The current shorthand mapping covers the main v1 inspector groups but does not attempt to fully decompose every CSS shorthand or semantic alias; future property expansion may need more mappings.

## Summary

The inspector now keeps its property groups visible while initializing only the selected authored class target’s values, and detached mode still restores the full combined element property view.

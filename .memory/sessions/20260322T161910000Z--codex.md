## Context

- The user requested implementation of the zero-step mounting plan for the public `hawk-eye` package.
- I reviewed the repo memory, public package entrypoints, build config, tests, and the existing Vite/plugin runtime split before editing.
- The current package has no CLI surface and no auto-mount path; the actual runtime UI still lives in `DesignTool` and must be wired into a consumer app.

## Actions

- Added `packages/hawk-eye/src/installer.ts` with an idempotent React + Vite installer that patches supported `vite.config.*` files and `src/main|index.(tsx|jsx)` entry files using `ts-morph`.
- Added a public CLI entry at `packages/hawk-eye/src/cli.ts`, wired the package `bin` and build entries, and shipped the new runtime dependency needed for AST-based patching.
- Updated the public and workspace READMEs to document `hawk-eye init` as the zero-step setup path while keeping the manual entrypoints available.
- Added `tests/installer.test.ts` to cover TypeScript and JavaScript app patching, idempotent re-runs, guarded failures, and manual fallback output; expanded `tests/smoke.test.ts` to assert the package bin declaration.
- Verified the change with `pnpm install`, `pnpm test -- --run tests/installer.test.ts tests/smoke.test.ts`, `pnpm type-check`, `pnpm build`, and a direct `node packages/hawk-eye/dist/cli.js --help` sanity check.

## Decisions

- Keep auto-mount implementation as an installer-time AST patch rather than changing the runtime architecture; the mounted UI remains the existing `DesignTool` behind a dev-only `import.meta.env.DEV` guard.
- Limit v1 automation to React + Vite projects with supported Vite config and React entry file shapes; unsupported or ambiguous projects receive a precise manual fallback message instead of a risky rewrite.

## Open Threads

- The public package still emits the existing `import.meta` warning in the CJS build because the browser runtime is re-exported into the package bundle; the new installer work does not worsen it, but it remains worth addressing separately if CJS polish matters.

## Summary

Added a public `hawk-eye init` installer CLI that patches supported React + Vite apps for zero-step mounting, documented the workflow, and covered it with installer-specific tests plus build/type-check verification.
